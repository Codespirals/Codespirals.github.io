Welcome!

This website is a small presentation of the codespirals project.

The project aims to set up a bunch of modular libraries that can be used to more easily create frequently used features of applications.
To achieve this the project modules commonly contain the following properties:

  • One or more "service" classes, which have to be registered in the app builder and will then be injected where required through standard DI.
  • An "options" class of some kind which will read a number of settings from a config file. These settings will be injected into the respective service.
  • An also to-be-registered and injected class implementing the basic ILogger interface.

Additionally, this space is used to experiment with basic HTML, CSS and JavaScript. Visual consistency and eye health not guaranteed.
